# Establish BrightLocal Rank Tracking Baseline

Set up rank tracking in BrightLocal for a new client and capture the Day 1 baseline keyword positions. This is the starting benchmark against which all future SEO progress is measured.

## Context

Every new client needs a rank tracking grid configured in BrightLocal before the SEO Specialist can begin regular monitoring. Without a baseline, there is no way to show the client measurable progress.

## Steps

1. **Create the client profile** in BrightLocal with master NAP data from the client brief.

2. **Configure the rank tracking grid:**
   - Add all target keywords from the client brief (minimum 5, ideally 10–20)
   - Set primary tracking location: main service city
   - Set secondary locations: any additional service cities from the client brief
   - Enable both Google Maps Pack and Google Organic tracking for all keywords
   - Set tracking frequency: daily

3. **Run the initial rank check.** Pull the first results. This may take up to 24 hours for BrightLocal to complete a full sweep.

4. **Save the baseline.** Export the initial rank data as the "Day 1 Baseline" — this will be referenced in every future report to show movement.

5. **Configure citation monitoring:** Enable BrightLocal's Citation Tracker for the client and run the first citation audit.

6. **Enable review monitoring:** Connect the client's Google Business Profile and Facebook Page to BrightLocal's review monitoring.

7. **Produce the BrightLocal setup confirmation** (format from `client-onboarding` skill Pillar 2), including the baseline rankings table for all target keywords.

8. **Deliver** to the Marketing Director and copy to the Reporting Director (for the first monthly report baseline).

## Deliverable

A BrightLocal setup confirmation including:
- Rank tracking grid configuration confirmed
- Day 1 baseline rankings for all target keywords (Maps + Organic)
- Initial citation audit results
- Review monitoring active confirmation

## Approval Gate

This is a read/setup task. No client-facing action is required. The output feeds the onboarding completion report.
