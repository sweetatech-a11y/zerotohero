# Client Onboarding Project

This project covers the complete workflow for onboarding a new client onto all four agency tool pillars. It begins when the board approves a new client (signed contract, kickoff call complete) and ends when the client is marked "active" and the specialist agents begin their regular monitoring cadences.

## Project Phases

### Phase 1: Kickoff (Day 1–2)
- Board approves new client
- CRM Director assigns the Onboarding Specialist
- Onboarding Specialist receives and reviews the client brief
- Any missing information is flagged and collected before setup begins

### Phase 2: Four-Pillar Setup (Day 2–7)
- **GHL sub-account:** Deployed from master snapshot, configured for client
- **BrightLocal:** Rank tracking grid + citation audit baseline
- **Awario:** Brand, competitor, and keyword streams configured
- **Ad accounts:** Meta Business Manager + Google Ads MCC access granted (if running ads)

### Phase 3: Onboarding Completion Report (Day 7–8)
- Onboarding Specialist compiles the full completion report
- CRM Director reviews
- Report queued for board approval
- Board approves → client marked "active"

### Phase 4: Handoff to Operations (Day 8–10)
- Lead Automation Specialist receives GHL sub-account for ongoing automation management
- SEO Specialist begins first GBP post and rank monitoring cycle
- Brand Monitoring Specialist runs first Awario sweep
- Reporting Director notes baseline data for first monthly report

## Success Criteria

- All four tool pillars fully configured and confirmed
- Board approval received
- All specialist agents have their starting baselines
- Client is active in the system within 10 business days of kickoff

## Notes

- This project should be instantiated as a new project instance for each new client, with the client's name in the project name (e.g., "Client Onboarding — Acme Plumbing").
- Do not mark the project complete until the board has explicitly approved the client as active.
- If any pillar cannot be completed due to missing client access, pause and flag to the CRM Director — do not mark incomplete setups as complete.
